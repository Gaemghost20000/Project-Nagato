1. Introduction

Purpose:

Define the vision and scope of a personal AI application that runs locally for everything except LLM inference.

Capture the motivations: seamless file management, conversational interface, web search/scraping, and advanced RAG workflows.

Background & Motivation:

Personal Use Case: You need a local-first environment to maintain privacy and control, with the flexibility to call remote models via OpenRouter or LiteLLM when required.

Model-Agnostic Access: Leveraging OpenRouter’s multi-vendor API or an open-source proxy (LiteLLM) to avoid vendor lock-in.

UX Requirements: Integrate a Warp-like TUI for CLI workflows and a Cursor-style code editor for file creation and editing.

Scope:

Core Modules: Chat Service, FileManager, Scraper Service, RAG Engine, SQL/Text2SQL Service.

Persistence: Store chat logs in JSONL for downstream fine-tuning and indexing; use SQLite (or flat files) for metadata and long-term storage.

Local-First: All microservices run on the user’s machine; only LLM calls go out to the cloud.

Audience & Stakeholders:

Primary: You (the developer/end-user) with strong technical expertise.

Secondary: Future collaborators who may extend or contribute to the project.
