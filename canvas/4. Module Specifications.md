4. Module Specifications

4.1 Chat Service

Purpose: Orchestrate conversational state, persistence, and integration with RAG/SQL.

Tech Stack: FastAPI (Python) or Express.js (Node).

Endpoints:

POST /chat — send user message; optional query params ?rag=true&sql=trueRequest: { "message": "...", "session_id": "..." }Response: { "response": "...", "session_id": "..." }

GET /history/{session_id} — retrieve JSONL transcriptResponse: newline-separated JSON entries: { "role": "user|assistant", "content": "..." }

POST /history/{session_id}/export — export to JSONL fileResponse: { "file_path": "..." }

POST /chat/stream — WebSocket/Server-Sent Events for token streaming

4.2 LLM Gateway

Purpose: Abstract providers (OpenRouter, LiteLLM) behind a unified client.

Tech Stack: Python SDK / Node SDK.

Interface:

gateway.send(prompt, options) — returns stream or full result.

Features:

Provider selection via config/env var

Rate-limiting and cost tracking

Retry logic and backoff

4.3 FileManager

Purpose: Expose filesystem operations to the UI and services.

Tech Stack: Node.js FS module or Python os/pathlib.

Endpoints/Commands:

list(path) — return file/directory listing

read(path) — return file contents

write(path, content) — create/update file

delete(path) — remove file/directory

Watch Mode: emit events on filesystem changes (via WebSocket or IPC)

Optional: Git integration for versioning

4.4 Scraper Service

Purpose: Fetch and preprocess web content for ingestion or direct chat.

Tech Stack: Python (Requests + BeautifulSoup) / Node.js (axios + cheerio).

Workflow:

Query: call SearxNG API (/search?q=...)

Fetch: use curl-based wrapper for raw HTML (with proxy/UA rotation)

Parse: extract main content, links, metadata

Return: { "title": "...", "text": "...", "links": [...] }

Endpoints:

GET /scrape?query=...&source=searx|curl

4.5 RAG Engine

Purpose: Index documents and retrieve relevant context for chat.

Tech Stack: Llama-Index (Python).

Components:

Indexer: ingest JSON, Markdown, PDFs

Vector Store: FAISS or SQLite+Embeddings

Retriever: top-k nearest neighbors

Endpoints:

POST /rag/index — upload document

POST /rag/query — { "query": "...", "top_k": 5 }Response: { "documents": [ {"id": "...","text": "..."} ] }

4.6 Text2SQL Service

Purpose: Convert natural language questions into SQL queries against local DB.

Tech Stack: LangChain’s text2sql or Datrics.

Process:

Receive NL query

Generate SQL via LLM

Execute against SQLite

Return results

Endpoint:

POST /sql/query — { "nl": "..." }Response: { "sql": "SELECT ...", "results": [ {...} ] }

4.7 UI Plugin Interfaces

Terminal Plugin:

Warp API: Define commands: ai chat, ai file read, ai scrape, etc.

Configuration: JSON manifest pointing to local service endpoints

Editor Plugin (VSCode):

Commands: “AI: Chat”, “AI: Insert File”, “AI: RAG Search”

Sidebar view for chat history and file browser

Webview or native panels for streaming responses
