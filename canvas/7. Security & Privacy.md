7. Security & Privacy

7.1 Data Residency & Encryption

Local-First Storage: All user data (chat logs, documents, indexes) stored locally on disk by default.

Optional Encryption: Allow users to enable AES-256 encryption with a passphrase for:

Chat transcripts (JSONL)

SQLite databases

Vector store files

7.2 Authentication & Access Control

API Key Management: Securely store and rotate LLM provider keys (OpenRouter, LiteLLM proxy) in an encrypted local vault.

User Authentication: Local user accounts or OS-level permissions to restrict access to the application data directory.

7.3 Network Security

Secure Transport: All remote calls (LLM Gateway, SearxNG over HTTP, provider APIs) use HTTPS with TLS.

Firewall Configuration: Recommend restricting inbound connections except for localhost; document necessary ports (e.g., 8000 for services).

7.4 Privacy

No Telemetry by Default: The app does not send usage analytics unless explicitly enabled.

Privacy Mode: Disable automatic web scraping or live RAG indexing if privacy concerns arise.

7.5 Compliance & Licensing

Open-Source Licenses: Verify that integrated dependencies (e.g., SearxNG, Llama-Index) comply with permissive licenses (Apache, MIT).

Data Compliance: If handling sensitive data, ensure GDPR best practices for data management (subject to local policy enforcement).

7.6 Audit & Logging

Secure Logs: Store audit logs of critical operations (e.g., key rotations, encryption toggles) in a write-once, tamper-evident format.

Log Rotation: Implement automatic log rotation to prevent unbounded disk growth.
