5. Technology Stack

Layer

Technology Options

Notes

Backend API

Python (FastAPI), Node.js (Express)

FastAPI for async Python RAG, Express for JS ecosystems

LLM Gateway

OpenAI SDK, LiteLLM SDK

Unified interface; LiteLLM for vendor-agnostic proxies

Database

SQLite, LevelDB, JSON/Flat Files

SQLite for structured metadata; flat files for simplicity

Vector Store

FAISS, SQLite + Embeddings

FAISS for high-performance; SQLite with vector extension

Scraper

SearxNG API, cURL + BeautifulSoup/Cheerio

SearxNG for meta-search; cURL fallback for raw fetch

RAG Framework

Llama-Index, LangChain

Llama-Index for indexing; LangChain for orchestration

Text2SQL

LangChain Text2SQL, Datrics

LangChain’s built-in; Datrics for lightweight SQL parsing

Frontend/UI

Electron, TUI (Ink/Blessed), VSCode Webview

Electron for cross-platform; Ink for native terminal UIs

Terminal Plugin

Warp Plugin API, Custom CLI (Go/Rust/Python)

Warp’s plugin spec or standalone CLI binary

Editor Plugin

VSCode Extension API, Theia Plugin

Cursor AI uses VSCode forks; Theia for open-source IDEs

Authentication

OAuth2, API Keys

Secure LLM access; local auth for plugins

Testing

Pytest, Jest

Unit + integration testing

CI/CD

GitHub Actions, GitLab CI

Automate builds, tests, and releases

Rationale

Python vs Node.js: Choose based on existing codebase and developer expertise. Python excels for ML/RAG workflows; Node.js integrates tightly with JS UIs.

SQLite: Zero-config and embeddable, ideal for local-first apps.

FAISS vs SQLite vectors: FAISS offers speed for large corpora; SQLite vector extension simplifies deployment.

Electron vs TUI: Electron gives full GUI; TUI offers lightweight, keyboard-driven workflows.

Warp vs Custom CLI: Leverage Warp for power-user features; fallback to custom CLI for portability.
