3. High-Level Architecture

3.1 Overview

The system is composed of loosely coupled microservices communicating over HTTP/IPC. All services run locally, except the LLM Gateway which proxies requests to remote providers (OpenRouter or LiteLLM).

3.2 Component Interaction

User Input: via Terminal UI or Editor Plugin

Chat Service: Receives input, optionally queries RAG and SQL Service, forwards to LLM Gateway

LLM Gateway: Routes to chosen provider, streams responses back

FileManager & Scraper: Invoked by Chat Service or directly by UI for file ops and web data ingestion

RAG Service: Indexes local docs and returns embeddings-based relevant chunks

SQL Service: Translates NL queries into SQL and executes against local DB

Persistence: Chat transcripts, index metadata, and DB files stored on disk

3.3 Data Flow

Chat Transcript: UI → Chat Service → JSONL store → RAG indexer

Web Scrape: UI → Scraper Service (SearxNG/cURL) → Parser → Local storage → RAG indexer

RAG Query: Chat Service triggers retriever → RAG Service returns chunks → appended context for LLM call

SQL Query: Chat Service sends NL query → SQL Service returns structured results → LLM summarization
