2. Goals & Requirements

2.1 High-Level Goals

Model Agnostic Access: Support OpenRouter and LiteLLM to switch between any remote LLM provider seamlessly.

Local-First Services: Run chat, file management, scraping, RAG indexing, and SQL locally to maximize privacy and control.

Seamless UX: Provide both a Warp-like terminal interface and a Cursor-style editor integration for code and file operations.

Scalable RAG Workflows: Enable efficient retrieval from large document collections using Llama‑Index and text2SQL.

Extensibility: Design modular components and plugin interfaces for future feature additions.

2.2 Functional Requirements

LLM Gateway: Adapter layer to route chat requests to OpenRouter or LiteLLM with authentication, rate‑limiting, and cost tracking.

Chat Service: REST/WebSocket API for interactive conversations, with endpoints for history export in JSONL.

FileManager: CRUD operations on local filesystem; watch mode for live file changes; optional Git integration.

Persistence: Store transcripts, metadata, and indexes in SQLite or flat‑file storage.

Scraper Service: Query SearxNG API; fallback to curl for raw page fetch; support proxies and user‑agent rotation.

RAG Engine: Build and maintain vector/text indexes via Llama‑Index; retrieve relevant chunks on demand.

Text2SQL Service: Convert natural language queries to SQL to query local document databases.

UI Plugins:

Terminal Plugin: Custom Warp plugin for invoking chat, scraping, file commands.

Editor Plugin: VSCode extension for inline chat, file creation/editing, and RAG-assisted code search.

2.3 Non-Functional Requirements

Performance: Low-latency local operations; configurable cache for RAG retrieval.

Security & Privacy: All user data stays on device except LLM calls; encrypt local storage if needed.

Cross‑Platform: Support Linux, macOS, and Windows (via WSL/Electron).

Reliability: Graceful degradation if offline (queuing LLM requests, local-only features still work).

Maintainability: Clear module boundaries, thorough documentation, and automated tests.

2.4 Constraints & Trade‑Offs

Bandwidth: Minimize external calls; batch RAG fetches to reduce API usage.

Resource Usage: Balance between index size and retrieval speed; configurable index parameters.

Open‑Source Licensing: Prefer permissive licenses for dependencies; be mindful of commercial SDK limits.

2.5 Acceptance Criteria

End‑to‑end demo of chat + file create/edit + export.

Working SearxNG and curl scraper with example query.

RAG query retrieving local docs via Llama‑Index.

Text2SQL translating NL to SQLite queries.

Basic Warp plugin and VSCode extension scaffolds.
